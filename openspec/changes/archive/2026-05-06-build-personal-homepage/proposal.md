## Why

The project needs a simple, public personal homepage that can be versioned on GitHub and deployed on Vercel without depending on a custom domain or backend service.

## What Changes

- Replace the default Vite starter screen with a complete personal homepage.
- Add responsive sections for identity, focus areas, projects, timeline, and contact links.
- Keep the site static so it can build to `dist/` and deploy cleanly on Vercel.
- Add project documentation and deployment notes for GitHub/Vercel handoff.

## Capabilities

### New Capabilities

- `personal-homepage`: Covers the static personal homepage experience, responsive layout, content sections, and deployment readiness.

### Modified Capabilities

- None.

## Impact

- Affects `src/main.ts`, `src/style.css`, `index.html`, static assets, README, and OpenSpec documentation.
- No runtime API, backend service, database, or private credentials are introduced.
